================================================================================
CONTEXTO SOBRE iSTAR 2.0
================================================================================

iStar 2.0 é uma notação para modelagem de requisitos orientada a objetivos.

ELEMENTOS PRINCIPAIS:

- ACTORS: Agentes do sistema (humanos, sistemas, organizações)
  Tipos: istar.Agent, istar.Role, istar.Actor

- GOALS: Objetivos que atores desejam alcançar
  Tipo: istar.Goal (dentro de nodes do ator)

- QUALITIES: Critérios de qualidade (ex: segurança, usabilidade)
  Tipo: istar.Quality (dentro de nodes do ator)

- TASKS: Atividades específicas para alcançar goals
  Tipo: istar.Task (dentro de nodes do ator)

- RESOURCES: Recursos necessários
  Tipo: istar.Resource (dentro de nodes do ator)

- DEPENDENCIES: Relações de dependência entre atores
  Um ator depende de outro para um elemento (goal, task, quality, resource)

- LINKS: Conexões entre elementos (refinamento, contribuição, etc.)

================================================================================
SUA TAREFA
================================================================================

Analise o cenário de requisitos acima e identifique áreas que precisam de 
clarificação para criar um modelo iStar 2.0 completo e preciso.

O cenário pode ter ambiguidades intencionais sobre:
- Quem são os atores do sistema
- Quais são os objetivos de cada ator
- Quais são os critérios de qualidade importantes
- Quais são as tarefas específicas
- Quais são as dependências entre atores

Sua função é gerar 5-8 perguntas de clarificação que ajudem a reduzir essas 
ambiguidades e permitam criar um modelo mais completo e preciso.

================================================================================
INSTRUÇÕES PARA AS PERGUNTAS
================================================================================

1. NÚMERO DE PERGUNTAS:
   - Gere entre 5 e 8 perguntas
   - Não mais, não menos

2. FOCO DAS PERGUNTAS:
   - Atores: Quem são os principais atores? Há atores implícitos?
   - Goals: Quais são os objetivos principais de cada ator?
   - Qualities: Quais são os critérios de qualidade (ex: segurança, usabilidade)?
   - Tasks: Quais são as tarefas específicas para alcançar os goals?
   - Dependencies: Quais são as dependências entre atores?
   - Resources: Quais recursos são necessários?

3. QUALIDADE DAS PERGUNTAS:
   - Cada pergunta deve ser ESPECÍFICA e ACIONÁVEL
   - Focada em um aspecto específico do modelo iStar
   - Capaz de ser respondida de forma concisa (1-3 frases)
   - Relevante para reduzir ambiguidade do cenário
   - Não genérica ou vaga

4. DISTRIBUIÇÃO:
   - Tente cobrir diferentes aspectos (atores, goals, tasks, etc.)
   - Não faça todas as perguntas sobre o mesmo tópico
   - Priorize ambiguidades que impactam mais o modelo

================================================================================
FORMATO DE SAÍDA
================================================================================

Liste as perguntas numeradas, uma por linha. Use este formato:

Q1: [Pergunta sobre atores]
Q2: [Pergunta sobre goals]
Q3: [Pergunta sobre qualities]
Q4: [Pergunta sobre tasks]
Q5: [Pergunta sobre dependencies]
Q6: [Pergunta adicional se necessário]
Q7: [Pergunta adicional se necessário]
Q8: [Pergunta adicional se necessário]

Cada pergunta deve:
- Começar com letra maiúscula
- Terminar com ponto de interrogação
- Ser clara e direta
- Focar em um aspecto específico

================================================================================
EXEMPLOS DE BOAS PERGUNTAS
================================================================================

BOM:
- "Quem são os três principais atores do sistema e qual é o papel de cada um?"
- "Quais são os dois objetivos principais do ator 'Passenger'?"
- "Existe algum critério de qualidade relacionado à segurança que deve ser considerado?"
- "Quais são as tarefas específicas que o ator 'Driver' deve realizar para aceitar uma corrida?"
- "O ator 'Passenger' depende do ator 'Driver' para qual elemento específico (goal, task ou resource)?"

RUIM:
- "Você pode me dizer mais sobre o sistema?" (muito genérica)
- "O que mais você sabe sobre isso?" (vaga)
- "Há mais alguma coisa importante?" (não específica)
- "Pode explicar tudo?" (muito ampla)

================================================================================

================================================================================

1. Analise o cenário cuidadosamente
2. Identifique as principais ambiguidades
3. Gere 5-8 perguntas específicas e acionáveis
4. Foque em aspectos que impactam o modelo iStar 2.0
5. Use o formato numerado (Q1, Q2, etc.)
6. Retorne APENAS as perguntas, sem texto adicional

Lembre-se: Suas perguntas devem ajudar a criar um modelo iStar 2.0 mais 
completo e preciso. Seja específico e focado.
