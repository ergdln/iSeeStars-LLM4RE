================================================================================
NOTAÇÃO iSTAR 2.0 - RESUMO
================================================================================

iStar 2.0 modela sistemas através de:

- ACTORS (Atores): Agentes do sistema (pessoas, sistemas, organizações)
  Tipos: istar.Agent, istar.Role, istar.Actor

- GOALS (Metas): Objetivos que atores desejam alcançar
  Tipo: istar.Goal (dentro de nodes do ator)

- TASKS (Tarefas): Atividades específicas para alcançar goals
  Tipo: istar.Task (dentro de nodes do ator)

- QUALITIES (Qualidades/Softgoals): Critérios de qualidade (ex: segurança, usabilidade)
  Tipo: istar.Quality (dentro de nodes do ator)

- RESOURCES (Recursos): Recursos necessários
  Tipo: istar.Resource (dentro de nodes do ator)

- DEPENDENCIES (Dependências): Relações entre atores
  Um ator depende de outro para um elemento (goal, task, quality, resource)

- LINKS (Conexões): Conexões entre elementos (refinamento, contribuição, etc.)

REGRAS CRÍTICAS
================================================================================

✓ tool = "pistar.2.0.0" (exatamente)
✓ istar = "2.0" (exatamente)
✓ Todos os IDs são UUIDs únicos em todo o modelo (formato UUID v4)
✓ source/target referem a IDs existentes
✓ Nodes sempre dentro de "nodes" de um ator
✓ TODOS os atores têm campo "customProperties": {"Description": ""}
✓ TODOS os nodes têm campo "customProperties": {"Description": ""}
✓ TODOS os dependencies têm campo "customProperties": {"Description": ""}
✓ Apenas atores, goals, tasks, qualities, resources e links
✓ JSON válido, sem comentários
✓ Coordenadas x, y são números (use valores apropriados, não apenas 0)

⚠️ REGRAS ESPECIAIS PARA DEPENDENCIES E LINKS:

1. DEPENDENCIES (array "dependencies"):
   - type DEVE ser: "istar.Goal" OU "istar.Task" OU "istar.Quality" OU "istar.Resource"
   - NUNCA use "istar.DependencyLink" como type de dependency!
   - source e target são IDs de ATORES (não nodes)
   - Representa: "ator source depende do ator target para um elemento do tipo type"

2. LINKS (array "links"):
   - type pode ser: "istar.DependencyLink", "istar.AndRefinementLink", etc.
   - source e target são IDs de NODES (elementos dentro de atores), NÃO atores!
   - Links conectam goals, tasks, qualities, resources entre si
   - Exemplo: um Goal pode ter um AndRefinementLink para Tasks que o refinam
