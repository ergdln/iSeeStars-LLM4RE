================================================================================
CRITÉRIOS DE VALIDAÇÃO
================================================================================

Execute as seguintes validações:

1. VALIDAÇÃO ESTRUTURAL
   - JSON é sintaticamente válido?
   - Todos os campos obrigatórios estão presentes?
   - tool = "pistar.2.0.0" (exatamente)?
   - istar = "2.0" (exatamente)?

2. VALIDAÇÃO DE IDs ÚNICOS
   - Todos os IDs são únicos em todo o modelo?
   - Não há IDs duplicados entre atores, nodes, dependencies ou links?
   - IDs são strings válidas (não vazias, não null)?
   - IDs são UUIDs no formato correto (xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)?

3. VALIDAÇÃO DE TIPOS DE ATORES
   - Tipos de atores são válidos?
   - Cada ator tem type = "istar.Agent" OU "istar.Role" OU "istar.Actor"?
   - Não há tipos inválidos ou malformados?

4. VALIDAÇÃO DE TIPOS DE ELEMENTOS (NODES)
   - Tipos de nodes são válidos?
   - Cada node tem type = "istar.Goal" OU "istar.Task" OU "istar.Quality" OU "istar.Resource"?
   - Não há tipos inválidos ou malformados?

5. VALIDAÇÃO DE TIPOS DE DEPENDENCIES
   - Tipos de dependencies são válidos?
   - Cada dependency tem type = "istar.Goal" OU "istar.Task" OU "istar.Quality" OU "istar.Resource"?
   - Não há tipos inválidos?

6. VALIDAÇÃO DE TIPOS DE LINKS
   - Tipos de links são válidos?
   - Tipos permitidos: AndRefinementLink, OrRefinementLink, ContributionLink, 
     DependencyLink, IsALink, ParticipatesInLink, QualificationLink, NeededByLink
   - Formato correto: "istar.[Tipo]" (ex: "istar.AndRefinementLink")?

7. VALIDAÇÃO DE REFERÊNCIAS (source/target)
   - Todos os source em dependencies referem a IDs de atores existentes?
   - Todos os target em dependencies referem a IDs de atores existentes?
   - Todos os source em links referem a IDs existentes (atores ou nodes)?
   - Todos os target em links referem a IDs existentes (atores ou nodes)?
   - Não há referências quebradas ou IDs inexistentes?

8. VALIDAÇÃO DE ESTRUTURA
   - Todos os atores têm: id, text, type, x, y, customProperties, nodes?
   - Todos os nodes têm: id, text, type, x, y, customProperties?
   - Todos os dependencies têm: id, text, type, x, y, customProperties, source, target?
   - Todos os links têm: id, type, source, target (label é opcional)?
   - Nodes estão dentro do array "nodes" de um ator (não em orphans)?

8.1. VALIDAÇÃO DE CUSTOM PROPERTIES (OBRIGATÓRIO)
   - TODOS os atores têm campo "customProperties"?
   - TODOS os nodes têm campo "customProperties"?
   - TODOS os dependencies têm campo "customProperties"?
   - Formato correto: {"Description": ""}?
   - Este campo é obrigatório para compatibilidade com ferramentas iStar

9. VALIDAÇÃO DE COMPATIBILIDADE iSTAR 2.0
   - Estrutura segue especificação iStar 2.0?
   - Relações entre elementos são válidas?
   - Dependencies conectam atores corretamente?
   - Links conectam elementos corretamente?

10. VALIDAÇÃO DE CAMPOS OBRIGATÓRIOS
    - "actors" existe e é array?
    - "orphans" existe e é array?
    - "dependencies" existe e é array?
    - "links" existe e é array?
    - "display" existe e é object?
    - "diagram" existe e tem: width, height, name, customProperties?

================================================================================
CLASSIFICAÇÃO DE SEVERIDADE
================================================================================

CRÍTICO: Problemas que tornam o JSON inválido ou não utilizável
- JSON sintaticamente inválido
- Campos obrigatórios faltando (tool, istar, actors)
- IDs duplicados
- Referências quebradas (source/target inexistentes)

ALTO: Problemas que afetam significativamente a estrutura
- Tipos inválidos
- Estrutura de elementos incorreta
- Campos obrigatórios de elementos faltando

MÉDIO: Problemas que podem causar problemas em uso
- Tipos malformados (ex: "Agent" ao invés de "istar.Agent")
- Campos opcionais faltando mas não críticos
- Valores inválidos em campos

BAIXO: Problemas menores ou sugestões
- Valores padrão que poderiam ser melhorados
- Estrutura que funciona mas poderia ser otimizada

================================================================================
EXEMPLOS DE PROBLEMAS E CORREÇÕES
================================================================================

EXEMPLO 1: ID Duplicado
Problema: Dois atores têm o mesmo ID "actor-001"
Sugestão: Alterar um dos IDs para "actor-002" ou usar IDs mais descritivos

EXEMPLO 2: Tipo Inválido
Problema: Ator tem type = "Agent" (deveria ser "istar.Agent")
Sugestão: Alterar para "istar.Agent"

EXEMPLO 3: Referência Quebrada
Problema: Dependency tem source = "actor-xyz" mas esse ator não existe
Sugestão: Verificar se o ID do ator está correto ou criar o ator faltante

EXEMPLO 4: Campo Faltante
Problema: Node não tem campo "x" ou "y"
Sugestão: Adicionar "x": 0, "y": 0 ao node

EXEMPLO 5: tool/istar Incorretos
Problema: tool = "pistar" ou istar = "2"
Sugestão: Alterar para tool = "pistar.2.0.0" e istar = "2.0"
