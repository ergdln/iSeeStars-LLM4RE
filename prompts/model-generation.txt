CONTEXTO SOBRE iSTAR 2.0
================================================================================

iStar 2.0 é uma notação para modelagem de requisitos orientada a objetivos, 
que permite representar sistemas através de atores, seus objetivos, tarefas, 
critérios de qualidade e dependências entre atores.

ELEMENTOS PRINCIPAIS:

1. ACTORS (Atores):
   - Representam agentes do sistema (humanos, sistemas, organizações)
   - Tipos válidos: istar.Agent, istar.Role, istar.Actor
   - Cada ator contém nodes (goals, tasks, qualities, resources)

2. GOALS (Metas):
   - Objetivos que atores desejam alcançar
   - Tipo: istar.Goal
   - Sempre associados a um ator (dentro do array nodes do ator)

3. QUALITIES (Softgoals/Critérios de Qualidade):
   - Objetivos qualitativos (ex: "segurança", "usabilidade", "performance")
   - Tipo: istar.Quality
   - Sempre associados a um ator (dentro do array nodes do ator)

4. TASKS (Tarefas):
   - Atividades específicas para alcançar goals
   - Tipo: istar.Task
   - Sempre associados a um ator (dentro do array nodes do ator)

5. RESOURCES (Recursos):
   - Recursos necessários para executar tarefas
   - Tipo: istar.Resource
   - Sempre associados a um ator (dentro do array nodes do ator)

6. DEPENDENCIES (Dependências):
   - Relações de dependência entre atores
   - Um ator (depender) depende de outro ator (dependee) para um elemento (dependum)
   - dependum pode ser Goal, Task, Quality ou Resource

7. LINKS (Conexões):
   - Conexões entre elementos (refinamento, contribuição, etc.)
   - Tipos: AndRefinementLink, OrRefinementLink, ContributionLink, etc.

REGRAS OBRIGATÓRIAS
================================================================================

1. ESTRUTURA:
   - tool DEVE ser exatamente "pistar.2.0.0"
   - istar DEVE ser exatamente "2.0"
   - Todos os campos obrigatórios devem estar presentes

2. IDs:
   - Todos os IDs devem ser strings únicas (use UUIDs)
   - Nenhum ID pode ser duplicado no modelo completo
   - source e target sempre referem a IDs existentes

3. ATOres:
   - Todos os atores no array "actors"
   - Cada ator tem array "nodes" contendo goals, tasks, qualities, resources
   - Tipos válidos: istar.Agent, istar.Role, istar.Actor

4. NODES:
   - Sempre dentro do array "nodes" de um ator
   - Tipos válidos: istar.Goal, istar.Task, istar.Quality, istar.Resource
   - Cada node tem id, text, type, x, y

5. DEPENDENCIES:
   - source: ID do ator que depende (depender)
   - target: ID do ator do qual depende (dependee)
   - type: Tipo do elemento dependido (Goal, Task, Quality, Resource)
   - text: Nome descritivo da dependência

6. LINKS:
   - Conectam elementos (nodes ou atores)
   - source e target referem a IDs existentes
   - type deve ser um tipo válido de link

7. CONTEÚDO:
   - Use informações do cenário original E das respostas
   - Incorpore detalhes fornecidos nas respostas
   - NÃO invente elementos não mencionados em nenhum dos dois
   - Se uma resposta fornece informação nova, use-a

8. FORMATO:
   - JSON válido, sem comentários
   - Aspas duplas para strings
   - Números para coordenadas x, y
